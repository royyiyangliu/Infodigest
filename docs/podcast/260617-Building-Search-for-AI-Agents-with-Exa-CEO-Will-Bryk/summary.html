为AI智能体重建搜索：Exa CEO Will Bryk 访谈

节目：AI + a16z  ·  嘉宾：Will Bryk（Exa 联合创始人兼CEO） ·  主持：Sarah Wang（a16z 合伙人） ·  时长：约49分钟  ·  总结日期：2026-06-17

本期访谈的核心论点：互联网时代的搜索是为人类设计的，而AI智能体的检索方式截然不同——它们需要更深的上下文、更完整的信息、处理远比"搜索框"复杂的问题。这一转变正在重塑信息检索的技术栈与商业格局。Exa 的定位是"为智能体而生的搜索引擎"，并自我定义为一家专注搜索的研究实验室。

一、起源与时机：为什么创业公司能在搜索上挑战 Google

  • Bryk 称搜索是其"人生使命"，高中起想做新型新闻机构，大学与联创 Jeff 用众包高质量链接做过原型搜索。
  • 转折点是 2021年：Transformer 变得足够强，"造一个比 Google 更好的搜索"首次成为可能，由此创立 Exa。
  • 关键"存在性证明"：对每个查询，把全网约一万亿份文档用 GPT-3 逐篇判断是否匹配、筛出前10条，效果会胜过 Google——但成本约 100亿美元/次查询。于是问题转化为纯粹的优化问题，方向上信奉"苦涩教训"（the bitter lesson）：数据越多，神经系统越强。
  • Exa 于 2022年11月（创业约一年半后）发布首个搜索引擎，Andrej Karpathy 转发、在推特走红；两周后 ChatGPT 发布，世界第一次意识到"在 Google 之外还有找信息的新方式"。
  • 2023年初开始有人索要 API，Exa 意识到可以服务这些"AI 产品/工作流"（当时还没有"智能体"一词）——而这些需求与 Bryk 自己当年想要的"深度搜索"高度一致。

二、智能体的搜索需求：与人类根本不同

Bryk 用比喻概括："智能体之于人类，犹如人类之于树懒"——为树懒造的外科医生不适合人类。智能体是"时间对其无意义"的新物种，要快速发起复杂查询、快速分析、为人类用户给出完美输出。他列出了约 20 处人类与智能体的差异，核心包括：

  • 复杂查询能力：不应强迫智能体把复杂语义压缩成关键词短语（会丢信息）；既要语义检索，也要关键词检索（如精确匹配化学式）。
  • 高度可控/可调：智能体有耐心叠加域名过滤、关键词过滤等"旋钮"，需要把底层开关全部暴露给它——这正是 Google 对人类不擅长的（"换几个词还是搜不对"）。
  • 结果要"全"：智能体不要10条或100条，而要 1,000 乃至 10,000 条。以投资人筛选生物科技公司为例：在做重大资金决策时不能漏掉任何一家潜在竞品。
  • 极端可定制的延迟：有的应用要超低延迟，有的完全不在乎延迟，需同时支持。
从零为智能体构建，会做出与传统搜索根本不同的架构决策——这是 Exa 区别于"老 Google / 新 Google（AI Mode）"的根基。Bryk 认为 Google 历次 I/O 的搜索改进，仍是围绕消费者快速答案优化，无法覆盖"找全所有竞品""按背景招聘 ML 工程师"等企业/智能体的深度复杂查询。

三、为什么小团队能造出"多方面优于 Google"的搜索

Exa 团队人数始终低于 100 人。Bryk 解释"既更容易、也更难"：

  • 更容易：Google 海量的人类点击数据对服务智能体"基本不重要"（仅作很小的排序信号）；LLM 让重排序（re-ranking）从"几百人团队"变成"几个人 + 自训小模型"即可。
  • 更难：客户需求越来越极致——传统搜索约 99.9% 可靠度，面向智能体则要 99.99%、99.999%。每多一个"9"在智能体经济里价值巨大，类比"Opus 4.6 已覆盖99.9%场景，但人人都要下一代"。

四、关键投资判断：LLM 比搜索更快被商品化

"很多知识工作本质上是搜索问题，而非智能问题。" Bryk 判断 LLM 正比搜索更快地被商品化：大部分知识工作并不需要最聪明的模型，"够大的开源模型 + 廉价的推理基础设施"即可胜任；只有发明新科学、数学、找代码 bug 等少数任务需要顶尖大模型。把知识工作画成难度同心圆，绝大部分面积已被现成模型覆盖。

对应到搜索：当你要为表格里每一个单元格补全"竞品/可招聘的人"时，每多一个"9"的检索质量都至关重要。最有价值、最易感知的场景是公司/人物搜索（go-to-market 情报）——几乎每家公司都要搜索"卖给谁、招聘谁"，且没人认为这是已解决的问题（所以工具不断被更换）。这也是 Exa 重点切入并内部自用（dog-fooding）的方向。

Bryk 进一步把搜索拔高为更宏大的命题："很多世界级难题其实是被包装过的搜索问题"——政治极化（信息不准确导致理性人变得不理性）、孤独（找不到同好/伴侣）都被他归为搜索问题。搜索是"围绕人类共同目标进行协调"的基础设施。

五、编码场景：Exa 已支撑 Devin

  • 编码智能体最需要最新鲜的信息（技术文档、SDK），否则每行代码都可能带致命错误；编码场景"智能很强，但检索质量还停留在2000年代初的黑暗时代"。
  • Cognition 测试后用 Exa 支撑 Devin，发现 Devin 明显更准、犯错更少。
  • 趋势判断：编码智能体与通用智能体将融合——编码 Agent 终将既搜代码、也搜世界信息与新闻。

六、"Token 末日"与检索的解法

针对"token apocalypse"（Uber 已用完年度服务预算、微软传出收回 Claude Code 许可证等），Bryk 的核心观点：检索能缓解 token 危机。不该对每个任务都用巨型模型，而应用"大模型决策 + 小模型执行"的模型家族；检索让小模型表现得像大模型。
  • Exa 通过只抽取文档中最相关的信息，使智能体少吃 token，相比其他供应商最高可为客户节省 20倍 成本。
  • 更激进的设想：模型权重不该浪费在记忆"法国首都"等世界知识上；未来可能出现 10亿参数甚至更小、"超级聪明但毫无知识（像没见过世界的爱因斯坦）"的模型，配合廉价工具调用——这是更高效、能缓解算力短缺的世界。
  • 时间判断：这一现实正在开始，到 2026年底 会非常明显（主持人认为这是"hot take"）。

七、研究方向与商业模式飞轮

  • 训练 LLM 的大思路同样适用于训练搜索模型：嵌入模型的预训练、后训练、以及在搜索工具上做 RL。
  • 博客实验：用 Tinker（Thinking Machines）在不同搜索工具上做 RL，发现"在 Exa 上做 RL"显著优于"包一层 Google SERP"——调用次数更少（更高效）且性能更高，因为 Exa 本就为智能体发起复杂查询而设计。
  • 研究遵循"苦涩教训"（多方向的 scaling laws），且大量研究未公开。研究动力来自业务："研究跟随需求"——超过 5,000 家企业客户每天从各个方向施压（更高质量、更快、更好的信息抽取）。
  • 评测困境：检索领域缺乏好的第三方 eval，现有基准不能代表智能体真实需求；客户的"ground truth"是自己的 A/B 测试与自建 eval——越成熟的客户越会自测，"一测就知道谁在顶部"。

八、市场规模与瓶颈演进

Bryk 预测：到 2039年，智能体搜索的市场规模将超过 Google 广告（在2030年代初期）。测算逻辑：LLM 调用数 × 需要搜索的比例 × 单次搜索成本，再外推趋势。人类平均每天搜索几次，而当人人都有个人助理、每个软件工具都用检索"自检工作"时，智能体的搜索量将是数百万量级的放大——"世界将像被电力填满一样被搜索填满"。

瓶颈的演进顺序：

  1. 基础设施（当下）：搜索量增长 10–1000 倍所需的基础设施"还没建出来"，需要超高吞吐的新型向量数据库。
  2. 数据：智能体会问"网上没有、甚至从未被记录"的世界信息——需要"发掘世界的数据"（如卫星图像、人脑中的信息），以理解作物产量如何影响某公司等深层问题。
  3. 检索：当网页从约一万亿页扩大到 千万亿（quadrillion） 页时，能在万亿页上运行的算法可能贵上千倍——"什么样的搜索算法能在千万亿页上运行"是几乎无人讨论的根本问题。

九、内容方与价值分配：把"做大蛋糕"留给生态

面对"网络日益封闭、数据方担心被'Stack Overflow 化'"的担忧，Bryk 给出第一性原理框架：智能体经济将极其庞大（智能体在赛博空间做商业、读信息，规模是人类活动的"千倍"）。

与其"几千亿美元流向一家公司"，不如例如 500亿/年 归一家、1500亿/年 归所有内容方；而总盘最终会是一万亿美元/年量级。设计出对内容生产者更友好的价值分配，能让大家都"做得极好"，并激励生产更多优质内容。（对内容/媒体资产的长期定价含义值得关注。）

十、文化、领导力与品牌细节（投资人视角的"软信号"）

  • SpaceX 经历：Bryk 曾在 SpaceX 实习，亲见首次海上回收着陆。他吸收了 Elon 的几点风格：极度关注细节（"老鹰隐喻"——高空盘旋、发现关键细节就俯冲深入再拉回）、善用"模因式命名"凝聚使命（如"让人类成为跨行星物种"）。他强调项目命名极重要，因为公司到一定规模后，名字能在他不在场的对话里锚定使命。
  • 品牌命名："Exa" 取 10^18（exa-前缀），对比 Google 的 10^100——寓意"提炼最重要的信息"而非用海量结果淹没用户。门口的"山羊（GOAT）"装置成了网红打卡点，已因此招到员工。
  • 文化：强调"激情/眼里有火"，让每个人做最想做的项目（"在搜索领域，任何方向的改进都对业务有利"），借 AI 工具并行推进多个项目。招聘上 Bryk 仍亲自面试每一位候选人，最看重 passion 与 agentic——"世界正变得极度精英化（meritocratic），因为智能体工具填补了能力鸿沟"。

十一、投资者关键启示

  1. "LLM 商品化快于搜索"是核心命题：若成立，价值会从"模型本身"向"检索/数据 + 编排"转移；以"大模型决策 + 检索增强的小模型执行"为主的架构，将压低单位 token 成本（Exa 称最高省20倍）。
  2. 智能体搜索是独立且高增长的赛道：需求特征（要全、可控、复杂查询、可调延迟）与人类搜索不同，构成对 Google 点击数据护城河的"绕行"；Bryk 给出 2039 年超 Google 广告的 TAM 判断。
  3. go-to-market 情报是最现实的变现锚点："找客户/找人"是普遍且未被解决的搜索问题，工具更换频繁，付费意愿强。
  4. 编码 Agent 是已验证的需求侧抓手：Exa 支撑 Devin 带来可量化的准确率提升，且编码与通用 Agent 将融合，打开横向扩张空间。
  5. 基础设施先于一切成为瓶颈：超高吞吐向量数据库、检索算法在"千万亿页"规模下的可扩展性，是潜在的卡脖子环节与投资机会。
  6. 内容/数据方的价值再分配：若智能体经济做到"一万亿/年"且向内容方倾斜分配，对媒体、数据供应商的长期商业模式与估值有重要含义。
  7. "研究实验室"型公司画像：小团队（<100人）、研究跟随 5000+ 企业的真实需求驱动，是评估此类基础设施公司壁垒时的重要软信号。
